VTK-7.0 + QT-5.5, OS X 10.11

How to compile vtk7.0.0 and QT5.5 in Mac OS X(EI Captain, 10.11.3)

Installing QT

  • Download QT with .dmg installer and install it (I used default)

Compiling and installing VTK

  • Download VTK source package .zip(from http://fossies.org/linux/misc/VTK-7.0.0.tar.gz/), and unpack it(or git it)
  • Install camke(I had CMake GUI and cmake installed from homebrew. I don’t know which tool I am really using. I guess the one from homebrew will do the work.)
  • It was rather complex with CMake GUI tool, so I decided to edit on the configuration file directly on the second time of compiling. (What, you ask why did I compiled twice? 😂I had an error testing with QT+VTK(Undefined symbols for architecture x86_64), and spent two days without solving it.. Finally I found this simple way to try my luck..)
  • Make a new directory under the path of the source folder, e.g. ./VTK is your source code folder, then run these commands in order:
    mkdir ./VTK/build
    cd ./VTK/build
    cmake ..

  • Now you shoud find a file ‘CMakeCache.txt’. Edit it!

    • Check VTK_Group_Qt. Run cmake ..again;
    • Later set the values related to QT to be ‘ON’(I don’t have theory behind, as I am a beginner😝. The reason I checked these values are I’m going to play around with QT any way, so they should be harmless),
    • Especially set the value: CMAKE_OSX_ARCHITECTURES=x86_64!(If you know what does it mean.)
    • I was lucky, that the compilition was successful after these blind settings. If you have this or that kind of errors, promise me, don’t feel upset. There’ll aways be a solution, only up to how long time would you like to spend on it 😃.

  • Go the build directory, you will find there are more file there. Simplly run

    make

    or

    make -j2 # the number denotes the processors you want to use

  • Afterwards install with 

    make install

Test

I always feel like to test after installing something, to see it the installation was alright. So, try this example! I really hope everything goes well with you installation, because I also don’t know what to do otherwise 😂.
