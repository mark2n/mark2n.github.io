How to compile vtk7.0.0 and QT5.5 in Mac OS X(EI Captain, 10.11.3)

Installing QT

  • Download QT with .dmg installer and install it (I used default)

Compiling and installing VTK

  • Download VTK source package .zip(from http://fossies.org/linux/misc/VTK-7.0.0.tar.gz/), and unpack it(or git it)
  • Install camke(I had CMake GUI and cmake installed from homebrew. I don‚Äôt know which tool I am really using. I guess the one from homebrew will do the work.)
  • It was rather complex with CMake GUI tool, so I decided to edit on the configuration file directly on the second time of compiling. (What, you ask why did I compiled twice? üòÇI had an error testing with QT+VTK(Undefined symbols for architecture x86_64), and spent two days without solving it.. Finally I found this simple way to try my luck..)
  • Make a new directory under the path of the source folder, e.g. ./VTK is your source code folder, then run these commands in order: 
    mkdir ./VTK/build
cd ./VTK/build
cmake ..

  • Now you shoud find a file ‚ÄòCMakeCache.txt‚Äô. Edit it!

    • Check VTK_Group_Qt. Run cmake ..again;
